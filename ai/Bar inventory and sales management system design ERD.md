## Convex Data Model Fundamentals

Convex is a document database with relational capabilities. Key conventions that
apply throughout this document:

- Every document automatically receives `_id` (typed as `Id<"tableName">`) and
  `_creationTime` (a `number` epoch ms timestamp). These are **not declared** in
  the schema — Convex injects them automatically.
- Cross-table references use `v.id("tableName")` — Convex's typed foreign key.
  There are no JOIN statements; related documents are fetched via `ctx.db.get(id)`
  or parallel lookups using `Promise.all`.
- Indexes are declared with `.index("name", ["field1", "field2"])` chained on
  the table definition and queried with `.withIndex()` in query/mutation functions.
- There is no `INT`, `VARCHAR`, or `DECIMAL`. Types are `v.string()`,
  `v.number()`, `v.boolean()`, `v.id()`, `v.optional()`, `v.union()`,
  `v.literal()`, `v.array()`, and `v.object()`.
- Timestamps are stored as `number` (milliseconds since epoch via `Date.now()`).
  Human-readable calendar dates are stored as ISO 8601 strings (e.g. `"2026-03-22"`).
- There is no auto-increment primary key. The `_id` field is Convex's globally
  unique, system-generated document identifier.
- Referential integrity is enforced in mutation functions, not at the database level.

## Tables and Documents

---

### 1. `bars`

Represents one of the physical bar outlets on the premises.

| Field           | Convex Type    | Notes                                  |
|-----------------|----------------|----------------------------------------|
| `_id`           | `Id<"bars">`   | Auto-generated by Convex               |
| `_creationTime` | `number`       | Auto-generated by Convex               |
| `name`          | `v.string()`   | Display name, e.g. "Pool Bar"          |
| `location`      | `v.string()`   | Physical location on premises          |
| `barType`       | `v.string()`   | e.g. `"main"`, `"vip"`, `"pool"`      |
| `isActive`      | `v.boolean()`  | Whether bar is currently operational   |

**Indexes:**

| Index Name    | Fields       | Purpose                              |
|---------------|--------------|--------------------------------------|
| `by_isActive` | `[isActive]` | Filter active bars for UI dropdowns  |

### 2. `beverages`

Represents a drink product tracked across the system.

| Field           | Convex Type       | Notes                                       |
|-----------------|-------------------|---------------------------------------------|
| `_id`           | `Id<"beverages">` | Auto-generated by Convex                    |
| `_creationTime` | `number`          | Auto-generated by Convex                    |
| `name`          | `v.string()`      | Display name, e.g. "Heineken 330ml"         |
| `category`      | `v.string()`      | e.g. `"beer"`, `"wine"`, `"spirit"`         |
| `unitOfMeasure` | `v.string()`      | e.g. `"bottle"`, `"can"`, `"litre"`         |
| `unitPrice`     | `v.number()`      | Selling price per unit                      |
| `reorderLevel`  | `v.number()`      | Minimum store quantity before alert fires   |
| `isActive`      | `v.boolean()`     | Whether beverage is currently sold          |

**Indexes:**

| Index Name    | Fields       | Purpose                                  |
|---------------|--------------|------------------------------------------|
| `by_category` | `[category]` | Filter/group beverages by category       |
| `by_isActive` | `[isActive]` | Exclude discontinued beverages           |

### 3. `users`

Represents system users — waiters, store managers, and administrators — who
access the platform. Supports SSO via an external auth provider.

| Field           | Convex Type              | Notes                                       |
|-----------------|--------------------------|---------------------------------------------|
| `_id`           | `Id<"users">`            | Auto-generated by Convex                    |
| `_creationTime` | `number`                 | Replaces `createdAt`; auto-generated        |
| `externalId`    | `v.string()`             | SSO / auth provider subject identifier      |
| `email`         | `v.string()`             | Unique user email                           |
| `name`          | `v.string()`             | Full name                                   |
| `phone`         | `v.optional(v.string())` | Contact phone number                        |
| `isActive`      | `v.boolean()`            | Active status flag                          |
| `lastLoginAt`   | `v.optional(v.number())` | Epoch ms timestamp of last login            |
| `updatedAt`     | `v.number()`             | Epoch ms of last mutation update            |

**Indexes:**

| Index Name      | Fields         | Purpose                                        |
|-----------------|----------------|------------------------------------------------|
| `by_externalId` | `[externalId]` | Auth lookup on login (SSO token → user doc)    |
| `by_email`      | `[email]`      | Unique email enforcement and lookup            |
| `by_isActive`   | `[isActive]`   | Filter active staff for shift assignment       |

### 4. `shifts`

Represents a working session for a user at a bar on a given date.

| Field           | Convex Type                | Notes                                    |
|-----------------|----------------------------|------------------------------------------|
| `_id`           | `Id<"shifts">`             | Auto-generated by Convex                 |
| `_creationTime` | `number`                   | Auto-generated by Convex                 |
| `userId`        | `v.id("users")`            | Reference to the working user            |
| `barId`         | `v.id("bars")`             | Reference to the bar                     |
| `shiftDate`     | `v.string()`               | ISO 8601 date, e.g. `"2026-03-22"`       |
| `startTime`     | `v.string()`               | ISO 8601 time, e.g. `"08:00:00"`         |
| `endTime`       | `v.optional(v.string())`   | Set when the shift is closed             |
| `isFinalized`   | `v.boolean()`              | Locks stock log entries when `true`      |

**Indexes:**

| Index Name       | Fields                | Purpose                                    |
|------------------|-----------------------|--------------------------------------------|
| `by_userId`      | `[userId]`            | All shifts for a given user                |
| `by_barId`       | `[barId]`             | All shifts at a given bar                  |
| `by_barId_date`  | `[barId, shiftDate]`  | Shifts at a bar on a specific date         |
| `by_userId_date` | `[userId, shiftDate]` | A user's shift on a specific date          |

### 5. `userStockLogs`

Per-beverage stock and sales reconciliation entry for a user within a shift.

| Field              | Convex Type            | Notes                                           |
|--------------------|------------------------|-------------------------------------------------|
| `_id`              | `Id<"userStockLogs">`  | Auto-generated by Convex                        |
| `_creationTime`    | `number`               | Auto-generated by Convex                        |
| `shiftId`          | `v.id("shifts")`       | Parent shift reference                          |
| `beverageId`       | `v.id("beverages")`    | Beverage being tracked                          |
| `openingStock`     | `v.number()`           | Units at start of shift                         |
| `newStockReceived` | `v.number()`           | Additional units collected during shift         |
| `totalStock`       | `v.number()`           | `openingStock + newStockReceived` (persisted)   |
| `closingStock`     | `v.number()`           | Physical count at end of shift                  |
| `salesQuantity`    | `v.number()`           | `totalStock − closingStock` (persisted)         |
| `salesValue`       | `v.number()`           | `salesQuantity × unitPrice` (persisted)         |
| `recordedAt`       | `v.number()`           | Epoch ms when entry was saved                   |

> Derived fields (`totalStock`, `salesQuantity`, `salesValue`) are stored rather
> than computed at query time to avoid expensive document fetches in Convex's
> query model.

**Indexes:**

| Index Name            | Fields                  | Purpose                                    |
|-----------------------|-------------------------|--------------------------------------------|
| `by_shiftId`          | `[shiftId]`             | All stock log entries for a shift          |
| `by_shiftId_beverage` | `[shiftId, beverageId]` | Single beverage lookup within a shift      |
| `by_beverageId`       | `[beverageId]`          | Cross-shift history for a beverage         |

| Field              | Convex Type              | Notes                                       |

| `_creationTime`    | `number`                 | Auto-generated by Convex                    |
| `beverageId`       | `v.id("beverages")`      | One inventory record per beverage           |
| `qtyInStore`       | `v.number()`             | Current quantity in central store           |
| `reorderThreshold` | `v.number()`             | Level at which a reorder alert fires        |
| `lastUpdated`      | `v.number()`             | Epoch ms of last stock movement             |

**Indexes:**

| Index Name      | Fields         | Purpose                                             |
|-----------------|----------------|-----------------------------------------------------|
| `by_beverageId` | `[beverageId]` | Unique lookup — one inventory doc per beverage      |

### 7. `storeTransactions`

Logs every movement of stock into or out of the central store.

| Field           | Convex Type                                              | Notes                                              |
|-----------------|----------------------------------------------------------|----------------------------------------------------|
| `_id`           | `Id<"storeTransactions">`                                | Auto-generated by Convex                           |
| `_creationTime` | `number`                                                 | Auto-generated by Convex                           |
| `beverageId`    | `v.id("beverages")`                                      | Beverage being transacted                          |
| `barId`         | `v.optional(v.id("bars"))`                               | Destination/source bar (`undefined` for supplier receive) |
| `userId`        | `v.optional(v.id("users"))`                              | User who initiated the transaction                 |
| `txnType`       | `v.union(v.literal("receive"), v.literal("issue"))`      | `"receive"` = supplier in; `"issue"` = bar/user out |
| `qty`           | `v.number()`                                             | Quantity transacted                                |
| `txnDate`       | `v.number()`                                             | Epoch ms timestamp                                 |
| `notes`         | `v.optional(v.string())`                                 | Optional reference or remarks                      |

**Indexes:**

| Index Name           | Fields                    | Purpose                                       |
|----------------------|---------------------------|-----------------------------------------------|
| `by_beverageId`      | `[beverageId]`            | Movement history for a beverage               |
| `by_barId`           | `[barId]`                 | Transactions belonging to a bar               |
| `by_userId`          | `[userId]`                | Collections made by a specific user           |
| `by_txnType`         | `[txnType]`               | Filter receives vs issues                     |
| `by_beverageId_date` | `[beverageId, txnDate]`   | Beverage movement over a date range           |

| Field          | Convex Type                                                                    | Notes                             |
|----------------|--------------------------------------------------------------------------------|-----------------------------------|

### 9. `salesSummaries`

Pre-aggregated sales data for fast multi-dimensional reporting. Populated by
scheduled Convex cron mutations rather than computed at query time.

| Field          | Convex Type                                                                                   | Notes                                               |
|----------------|-----------------------------------------------------------------------------------------------|-----------------------------------------------------|
| `_id`          | `Id<"salesSummaries">`                                                                        | Auto-generated by Convex                            |
| `_creationTime`| `number`                                                                                      | Auto-generated by Convex                            |
| `barId`        | `v.id("bars")`                                                                                | Bar this summary covers                             |
| `userId`       | `v.optional(v.id("users"))`                                                                   | `undefined` for bar-level rollups                   |
| `beverageId`   | `v.id("beverages")`                                                                           | Beverage this summary covers                        |
| `periodType`   | `v.union(v.literal("daily"), v.literal("weekly"), v.literal("monthly"), v.literal("yearly"))` | Aggregation granularity                             |
| `periodKey`    | `v.string()`                                                                                  | `"2026-03-22"` / `"2026-W12"` / `"2026-03"` / `"2026"` |
| `year`         | `v.number()`                                                                                  | Calendar year (enables year-on-year comparisons)    |
| `month`        | `v.optional(v.number())`                                                                      | Month 1–12 (monthly and yearly records only)        |
| `weekNumber`   | `v.optional(v.number())`                                                                      | ISO week number (weekly records only)               |
| `totalQtySold` | `v.number()`                                                                                  | Total units sold in the period                      |
| `totalRevenue` | `v.number()`                                                                                  | Total revenue in the period                         |

**Indexes:**

| Index Name                  | Fields                                         | Purpose                                      |
|-----------------------------|------------------------------------------------|----------------------------------------------|
| `by_barId_period`           | `[barId, periodType, periodKey]`               | Bar performance over a specific period       |
| `by_userId_period`          | `[userId, periodType, periodKey]`              | User (waiter) performance over a period      |
| `by_beverageId_period`      | `[beverageId, periodType, periodKey]`          | Beverage trend over a period                 |
| `by_barId_beverage_period`  | `[barId, beverageId, periodType, periodKey]`   | Bar + beverage breakdown                     |
| `by_year_periodType`        | `[year, periodType]`                           | Year-on-year comparisons                     |

## Full Convex Schema (`convex/schema.ts`)

```typescript
import { defineSchema, defineTable } from "convex/server";
import { v } from "convex/values";

export default defineSchema({

  bars: defineTable({
    name: v.string(),
    location: v.string(),
    barType: v.string(),
    isActive: v.boolean(),
  })
    .index("by_isActive", ["isActive"]),

  beverages: defineTable({
    name: v.string(),
    category: v.string(),
    unitOfMeasure: v.string(),
    unitPrice: v.number(),
    reorderLevel: v.number(),
    isActive: v.boolean(),
  })
    .index("by_category", ["category"])
    .index("by_isActive", ["isActive"]),

  users: defineTable({
    externalId: v.string(),
    email: v.string(),
    name: v.string(),
    phone: v.optional(v.string()),
    isActive: v.boolean(),
    lastLoginAt: v.optional(v.number()),
    updatedAt: v.number(),
  })
    .index("by_externalId", ["externalId"])
    .index("by_email", ["email"])
    .index("by_isActive", ["isActive"]),

  shifts: defineTable({
    userId: v.id("users"),
    barId: v.id("bars"),
    shiftDate: v.string(),
    startTime: v.string(),
    endTime: v.optional(v.string()),
    isFinalized: v.boolean(),
  })
    .index("by_userId", ["userId"])
    .index("by_barId", ["barId"])
    .index("by_barId_date", ["barId", "shiftDate"])
    .index("by_userId_date", ["userId", "shiftDate"]),

  userStockLogs: defineTable({
    shiftId: v.id("shifts"),
    beverageId: v.id("beverages"),
    openingStock: v.number(),
    newStockReceived: v.number(),
    totalStock: v.number(),
    closingStock: v.number(),
    salesQuantity: v.number(),
    salesValue: v.number(),
    recordedAt: v.number(),
  })
    .index("by_shiftId", ["shiftId"])
    .index("by_shiftId_beverage", ["shiftId", "beverageId"])
    .index("by_beverageId", ["beverageId"]),

  storeInventory: defineTable({
    beverageId: v.id("beverages"),
    qtyInStore: v.number(),
    reorderThreshold: v.number(),
    lastUpdated: v.number(),
  })
    .index("by_beverageId", ["beverageId"]),

  storeTransactions: defineTable({
    beverageId: v.id("beverages"),
    barId: v.optional(v.id("bars")),
    userId: v.optional(v.id("users")),
    txnType: v.union(v.literal("receive"), v.literal("issue")),
    qty: v.number(),
    txnDate: v.number(),
    notes: v.optional(v.string()),
  })
    .index("by_beverageId", ["beverageId"])
    .index("by_barId", ["barId"])
    .index("by_userId", ["userId"])
    .index("by_txnType", ["txnType"])
    .index("by_beverageId_date", ["beverageId", "txnDate"]),

  reorderAlerts: defineTable({
    beverageId: v.id("beverages"),
    qtyAtAlert: v.number(),
    reorderLevel: v.number(),
    alertedAt: v.number(),
    status: v.union(
      v.literal("open"),
      v.literal("acknowledged"),
      v.literal("resolved")
    ),
  })
    .index("by_beverageId", ["beverageId"])
    .index("by_status", ["status"])
    .index("by_beverageId_status", ["beverageId", "status"]),

  salesSummaries: defineTable({
    barId: v.id("bars"),
    userId: v.optional(v.id("users")),
    beverageId: v.id("beverages"),
    periodType: v.union(
      v.literal("daily"),
      v.literal("weekly"),
      v.literal("monthly"),
      v.literal("yearly")
    ),
    periodKey: v.string(),
    year: v.number(),
    month: v.optional(v.number()),
    weekNumber: v.optional(v.number()),
    totalQtySold: v.number(),
    totalRevenue: v.number(),
  })
    .index("by_barId_period", ["barId", "periodType", "periodKey"])
    .index("by_userId_period", ["userId", "periodType", "periodKey"])
    .index("by_beverageId_period", ["beverageId", "periodType", "periodKey"])
    .index("by_barId_beverage_period", ["barId", "beverageId", "periodType", "periodKey"])
    .index("by_year_periodType", ["year", "periodType"]),

});
```
